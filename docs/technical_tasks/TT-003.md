# Technical Task: Wire Code Review into Delivery Pipelines

## Overview

| | |
|---|---|
| **Task ID** | TT-003 |
| **Task Name** | Wire Code Review into Delivery Pipelines |
| **Category** | Infrastructure |
| **Goal** | `code-review/SKILL.md` exists but is never invoked automatically by `deliver-use-case` or `resolve-bug` — those pipelines only check spec conformance (`evaluate`) and E2E coverage. Add an isolated `code-review` subagent and wire a Code Review Gate into both pipelines so security/maintainability review runs automatically, not just on manual `/code-review` invocation. |
| **Status** | Done |

## Acceptance Criteria

- [x] `nexa-claude-core/agents/code-review.md` exists, mirroring the isolation pattern of `nexa-claude-core/agents/evaluate.md`
- [x] `deliver-use-case` runs a Code Review Gate (up to 2 iterations) as part of its verification stage, alongside the existing `evaluate` gate
- [x] `resolve-bug` runs the same Code Review Gate on the fix before closing the loop
- [x] Failure/iteration handling for Code Review Gate failures matches the existing pattern for `evaluate`/`DoD_BUG` failures (iteration cap of 2, rollback via saved commit hash on exhaustion)
- [x] Both pipelines' Completion summary tables include a "Code Review" row

## Affected Areas

- `nexa-claude-core/agents/code-review.md` (new)
- `nexa-claude-nextjs/skills/deliver-use-case/SKILL.md`
- `nexa-claude-nextjs/skills/resolve-bug/SKILL.md`

## Dependencies

- None

## Decisions

| Decision | Provenance | Source/Reasoning |
|----------|------------|------------------|
| New agent wrapper mirrors `agents/evaluate.md` exactly (frontmatter shape, "operating manual" pointer, restated DO NOT rules, Output Format passthrough) | EXPLICIT | Confirmed pattern by reading `nexa-claude-core/agents/evaluate.md` directly — this is the only existing typed-agent template in nexa-claude-core. |
| Code Review Gate placed after DoD/DoD_BUG passes, before Post-Implementation Tracking runs, in both pipelines | INFERRED (corrected via code review) | `resolve-bug` already deferred `implement/SKILL.md`'s tracking section explicitly; `deliver-use-case` did not — its Step 2 ran `implement/SKILL.md` (including its tracking/commit/close) unconditionally before the new DoD/Code Review gates. Added the same explicit deferral to `deliver-use-case` Step 2 so the gate actually reviews before commit/close, not after. |
| Iteration cap of 2, rollback via saved commit hash | EXPLICIT | Reuses the existing convention already applied to every other verification gate in both pipelines — no new failure-handling vocabulary introduced. |
| `code-review/SKILL.md`'s documented `$ARGUMENTS` scope extended to include `BUG-XXX` | INFERRED (corrected via code review) | The new agent wrapper's role text already claimed BUG-XXX support (needed since `resolve-bug` invokes it on bug fixes), but the underlying SKILL.md only documented UC/TT — a real scope mismatch for any future caller relying on the SKILL.md alone. |
