An interactive guide to applying Test-Driven Development using Vitest for unit tests and Playwright for end-to-end tests.

💡 The Core Question

Does TDD make sense for both Vitest (Unit) and Playwright (E2E)?

Yes, but they operate on entirely different feedback loops. TDD is famously associated with the rapid, millisecond feedback of Unit Tests (Vitest). However, TDD also applies to E2E tests (Playwright) through a methodology known as Acceptance Test-Driven Development (ATDD) or Behavior-Driven Development (BDD).

🧪 Vitest (The Inner Loop)

Highly granular. You write a failing test for a specific Next.js component, hook, or utility function. You make it pass, then refactor. The feedback loop is almost instantaneous.

🎭 Playwright (The Outer Loop)

Broad scope. You write a failing E2E test defining a complete user journey (e.g., "User logs in"). It drives the overarching feature requirements. The feedback loop is slower but guarantees actual business value.

Key Takeaways

  • ■
    Execution Speed Vitest runs in milliseconds, making it ideal for the rapid Red-Green-Refactor loop. Playwright launches browsers, making it too slow for minute-by-minute TDD.
  • ■
    Confidence Level Playwright gives maximum confidence that the application actually works for the user. Vitest only proves isolated components work technically.
  • ■
    TDD Iteration Suitability You should spend 80% of your TDD time in Vitest (Inner Loop) building the blocks, and 20% in Playwright (Outer Loop) verifying the assembled blocks.
