# Use Case: Calculate Sum

|             |                                                                      |
|-------------|----------------------------------------------------------------------|
| Use Case ID | UC-001                                                               |
| Use Case Name | Calculate Sum                                                        |
| Primary Actor | Visitor                                                              |
| Goal        | Enter two positive natural numbers and see their sum                 |
| Status      | Approved                                                             |

## Preconditions

- The calculator page is loaded in the browser.

## Main Success Scenario

1. Visitor sees the calculator form with two empty input fields, a "Calculate" button, and a "Clear" button.
2. Visitor enters a positive natural number into the first input field.
3. Visitor enters a positive natural number into the second input field.
4. Visitor clicks the "Calculate" button.
5. System validates both inputs.
6. System calculates the sum of the two numbers.
7. System displays the result below the form.

## Alternative Flows

### A1: Empty Input

**Trigger:** Visitor clicks "Calculate" with one or both fields empty (step 4).
**Flow:**

1. System displays an error message indicating which field(s) are empty.
2. System does not calculate or display a result.
3. Visitor corrects the input. Use case continues at step 4.

### A2: Non-Numeric Input

**Trigger:** Visitor enters letters, symbols, or other non-numeric characters (step 5).
**Flow:**

1. System displays an error message indicating the input must be a number.
2. System does not calculate or display a result.
3. Visitor corrects the input. Use case continues at step 4.

### A3: Zero or Negative Number

**Trigger:** Visitor enters zero or a negative number (step 5).
**Flow:**

1. System displays an error message indicating the input must be a positive natural number (1 or greater).
2. System does not calculate or display a result.
3. Visitor corrects the input. Use case continues at step 4.

### A4: Decimal Number

**Trigger:** Visitor enters a decimal number (step 5).
**Flow:**

1. System displays an error message indicating the input must be a whole number.
2. System does not calculate or display a result.
3. Visitor corrects the input. Use case continues at step 4.

### A5: Number Exceeds 10 Digits

**Trigger:** Visitor enters a number with more than 10 digits (step 5).
**Flow:**

1. System displays an error message indicating the maximum allowed is 10 digits.
2. System does not calculate or display a result.
3. Visitor corrects the input. Use case continues at step 4.

### A6: Clear Inputs

**Trigger:** Visitor clicks the "Clear" button at any time.
**Flow:**

1. System clears both input fields.
2. System clears any displayed result or error message.
3. Use case restarts at step 1.

## Postconditions

### Success Postconditions

- The sum of the two entered numbers is displayed to the visitor.
- Both input fields retain their entered values.

### Failure Postconditions

- No result is displayed.
- An error message is displayed indicating what is wrong.
- The input fields retain the visitor's entered values so they can correct the error.

## Business Rules

### BR-001: Valid Input Range

Each operand must be a positive natural number (integer ≥ 1) with a maximum of 10 digits (up to 9,999,999,999).

### BR-002: No Persistence

The calculation exists only in memory for the current interaction. Nothing is stored or saved (C-002).

### BR-003: Compute on Submit

The sum is calculated only when the visitor explicitly clicks the "Calculate" button, not in real-time as inputs change.
