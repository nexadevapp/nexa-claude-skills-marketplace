# Use Case: Redirect to destination

## Overview

| | |
|---|---|
| **Use Case ID** | UC-003 |
| **Use Case Name** | Redirect to destination |
| **Primary Actor** | Anonymous Visitor (anyone with a short link) |
| **Goal** | Resolve a short link and send the visitor to the destination URL |
| **Status** | Approved |
| **Functional Requirements** | FR-003 |

## Preconditions

- The visitor has a short link of the form `https://lnk.sh/{slug}`.
- The shortener service is reachable.

## Main Success Scenario

1. Visitor's user agent issues `GET /{slug}` to the lnk.sh origin.
2. System looks up the `LINK` row where `LINK.slug = {slug}`.
3. System verifies the link is not expired: either `LINK.expiresAt IS NULL`, or `LINK.expiresAt > now()`.
4. System verifies the destination is not currently on the abuse blocklist.
5. System issues an HTTP `302 Found` response with `Location: {LINK.destinationUrl}`, `Cache-Control: private, no-store`.
6. The user agent follows the redirect to the destination URL.

## Alternative Flows

### A1: Slug not found

**Trigger:** step 2 returns no row.
**Flow:**

1. System renders an HTTP `404 Not Found` page styled per the wireframe `#UC-003` failure state.
2. The page offers a CTA "← Back to lnk.sh" that links to UC-002.
3. Use case ends in failure.

### A2: Link expired

**Trigger:** step 3 finds `LINK.expiresAt <= now()`.
**Flow:**

1. System renders an HTTP `410 Gone` page with the message "This link has expired" and the explanation that anonymous links expire after 30 days.
2. The page offers a CTA to UC-002.
3. Use case ends in failure.

### A3: Destination is currently blocked

**Trigger:** step 4 finds the destination on the blocklist (a destination may be added to the blocklist after a link was created).
**Flow:**

1. System renders an HTTP `451 Unavailable For Legal Reasons` page with a generic message: "This link is no longer available."
2. The page does **not** disclose the destination URL, the rule matched, or the report source.
3. Use case ends in failure.

### A4: Repeated request from the same client

**Trigger:** step 1 occurs and a recent identical `GET /{slug}` is in cache.
**Flow:**

1. System may serve a cached response with the same `Location` header, provided the underlying `LINK` has not expired since the cache entry was issued.
2. Use case continues at step 6.

## Postconditions

### Success Postconditions

- The user agent receives a 302 with the correct `Location`.
- No `LINK` row is mutated. The redirect is idempotent.
- No tracking pixel, no third-party request, no analytics beacon fires (NFR-008).

### Failure Postconditions

- The visitor sees an actionable error page (404, 410, or 451).
- No information is disclosed about why a destination was blocked beyond the generic message.

## Business Rules

### BR-001: Privacy-respecting redirect

The redirect response must include `Cache-Control: private, no-store` and must not include `Set-Cookie`, third-party script tags, or analytics pixels. The redirect is the only side effect.

### BR-002: No interstitial page on success

The system never shows an interstitial "Redirecting…" page on the success path. The HTTP 302 is the response.

### BR-003: 410 vs 404

A slug that **once existed** but has since expired returns `410 Gone`. A slug that has **never existed** returns `404 Not Found`. The distinction is observable to the user agent and is intentional — it tells the difference between "this used to work" and "this was never valid."

### BR-004: Generic blocklist message

The blocklist response (A3) must use the same template regardless of the rule matched. The visitor cannot determine, by reading the response, why a particular destination was rejected.

## Non-Functional

- p95 redirect latency < 100ms over a rolling 7-day window (NFR-002).
- The slug lookup must use the unique index on `LINK.slug` (NFR-003) — never a table scan.
- All HTTP traffic must be served over TLS 1.3 (NFR-006).

## Traceability

- **Requirements:** FR-003 (Redirect), NFR-002 (Redirect latency), NFR-003 (Slug uniqueness), NFR-006 (TLS), NFR-008 (No third-party tracker).
- **Entities:** reads `LINK` by `slug`; never writes.
- **Wireframe:** [`docs/wireframes/index.html#UC-003`](../wireframes/index.html#UC-003) — failure-state UI only; the success path has no UI.
