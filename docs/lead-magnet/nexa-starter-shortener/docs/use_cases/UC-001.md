# Use Case: List my links

## Overview

| | |
|---|---|
| **Use Case ID** | UC-001 |
| **Use Case Name** | List my links |
| **Primary Actor** | Link Owner |
| **Goal** | View all links the owner has created so they can find and copy any of them |
| **Status** | Approved |
| **Functional Requirements** | FR-004 |
| **Dependencies** | TT-001 (Auth middleware) |

## Preconditions

- The owner is authenticated; a session exists for the current `USER`.
- The owner has navigated to `/dashboard`.

## Main Success Scenario

1. Owner navigates to `/dashboard`.
2. System verifies the session via the auth middleware (TT-001) and resolves the current `USER`.
3. System reads all `LINK` rows where `ownerId = currentUser.id`, ordered by `createdAt DESC`.
4. System computes summary stats: total links, owned (permanent) count, links expiring within 7 days.
5. System renders the dashboard with: a stats strip, a filter bar, and a table of links showing `slug`, `destinationUrl` (truncated), status (Owned · `Nd left`), and `createdAt`.
6. Owner clicks the `slug` in any row.
7. System copies the full short URL (`https://lnk.sh/{slug}`) to the owner's clipboard and shows a transient confirmation.

## Alternative Flows

### A1: Owner has no links yet

**Trigger:** step 3 returns zero rows.
**Flow:**

1. System displays an empty-state panel with the message "You haven't shortened any links yet" and a primary CTA "+ Shorten your first link" linking to UC-002.
2. Use case ends in success (the empty state is a valid result).

### A2: Filter by status

**Trigger:** owner clicks "Owned" or "Anonymous" in the filter bar (step 5).
**Flow:**

1. System refines the result set to links matching the selected status.
   - "Owned" = `ownerId IS NOT NULL` (always the current user, by step 3 query).
   - "Anonymous" = links the owner has *claimed* — out of scope for this release; the filter is rendered but disabled.
2. The table updates without a full page reload.
3. Use case continues at step 6.

### A3: Free-text filter

**Trigger:** owner types into the filter input (step 5).
**Flow:**

1. System filters the rendered list to rows where `slug` or `destinationUrl` contains the typed substring (case-insensitive).
2. The filter applies client-side over the already-loaded set.
3. Use case continues at step 6.

### A4: Pagination — "Load more"

**Trigger:** owner has more than the initial page size (default 20) of links and clicks "Load more" (step 5).
**Flow:**

1. System fetches the next page of `LINK` rows, ordered by `createdAt DESC`.
2. New rows are appended to the table.
3. Use case continues at step 6.

### A5: Session expired

**Trigger:** auth middleware rejects the session at step 2.
**Flow:**

1. System redirects the visitor to the sign-in page (provided by TT-001).
2. Use case ends in failure for this attempt.

## Postconditions

### Success Postconditions

- The owner sees their links, sorted by recency.
- No state is mutated. The dashboard is read-only in this use case.

### Failure Postconditions

- The owner is redirected to sign in (A5), or sees an empty-state panel (A1) — both treated as recoverable.

## Business Rules

### BR-001: Visibility scope

An owner sees **only their own links**. Even if a link's slug is known to others, no owner can list links they do not own. The query must always filter by `ownerId = currentUser.id`.

### BR-002: No deletion in this release

The `⋯` action menu shown next to each row contains "Copy link" only — no delete, no edit, no transfer. Destructive operations are out of scope for this release.

### BR-003: Expiry indicator

Links nearing `expiresAt` (within 7 days) are flagged with a warm-tone status pill ("3d left"). Owned links display "Owned" and never an expiry pill.

## Non-Functional

- Page interactive within 500ms p95 on a fresh session (cold start tolerance).
- The links query must be index-backed by `(ownerId, createdAt DESC)` on `LINK`.
- WCAG 2.1 AA: the table must be keyboard-navigable; status pills must not rely on color alone (NFR-009).

## Traceability

- **Requirements:** FR-004 (List owned links), NFR-009 (WCAG).
- **Entities:** reads `LINK` filtered by `ownerId`; reads `USER` for the navbar.
- **Dependencies:** TT-001 — Auth middleware (delivered by `/setup-web-middleware`).
