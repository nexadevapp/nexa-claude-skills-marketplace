No HTML body. No tracking pixel. No Set-Cookie. Total wire time target: < 100ms p95.
BR-001 — privacy-respecting: Response body is empty. The browser never renders an interstitial because the server never produces one. Any analytics for this redirect happen server-side, not in the browser.
BR-002 — no interstitial: Even if a future stakeholder asks for "redirecting…" text, the answer is no. Latency budget plus privacy posture forbid it.

Distinguished from 410: 404 is for slugs that never existed. The user agent treats this differently from 410 (gone) — bookmarks, search engines, and Linkrot crawlers see "this was never valid" instead of "this used to work."
Path A2 · 410 Gone — anonymous link expired
410
This link has expired

Anonymous links expire 30 days after they're created. Shorten a new one — or sign in to keep links forever.

+ Shorten a new link Sign in
Conversion moment: 410 is the second strongest sign-up moment after the post-shorten success state. The user just hit a dead link they cared about — the offer "sign in to keep links forever" is timely.
Status code matters: 410 (vs 404) tells crawlers and clients that this URL used to work. SEO and bookmark tools handle the two differently.
Path A3 · 451 Unavailable — destination on blocklist

If you believe this is a mistake, please contact support.

← Back to lnk.sh
BR-004 — generic message: The page deliberately omits any mention of which destination, which rule, or which reporter triggered the block. Disclosure would let bad actors probe the blocklist.
Status code 451: Honest signal to clients that the resource exists but cannot be served for policy reasons. More accurate than 404 (which would imply non-existence) or 403 (which implies the requester specifically is forbidden).
